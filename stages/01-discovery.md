# Stage 1 — Discovery

**The question:** Am I building the *right* thing?

## What this stage is

Turning a raw idea into a clear, bounded intent *before* any code exists. What it does, what it explicitly does **not** do, and what "done" looks like.

## Why it matters more when solo

You have no PM to push back. Most solo waste happens here: skip it, build for three hours, then realize the shape was wrong. This is the cheapest place to be wrong — a sentence is free to rewrite; code is not.

## Key practices

- Write the intent down as prose *before* touching code. If you can't write it, you don't understand it yet.
- Define non-goals explicitly. Scope creep is a solo builder's slow death.
- State the "done" condition as an observable behavior, not a feeling.
- Research real decisions instead of guessing (library choice, API shape, market question).

## Tooling

- `brainstorming` skill — forces intent → spec. Your "PM."
- `feature-boilerplate` / `plan-feature` / `quick-stories` — idea → scoped spec + GitHub epic with sized, prioritized user stories.
- `deep-research` skill — fan-out, fact-checked research for genuine decisions.
